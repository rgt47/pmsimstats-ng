# manuscripts/

Journal-format manuscripts built on the pmsimstats-ng codebase.

Organised following the `res/04-runin-power-analysis` pattern: a
shared compendium (this repository) with per-paper subdirectories
under `manuscripts/`. Each subdirectory holds a single paper's
`.Rmd`/`.tex`/`.pdf`, paper-specific `references.bib`, and any
CSL style file; shared numerical assets live in the sibling
`data/`, `figures/`, and `tables/` directories.

## Conventions

- **Per-paper subdirectory** named `NN-short-slug/` with entry
  point `report.Rmd` that knits to `report.tex` and `report.pdf`.
- **Paper-specific `references.bib`** inside each subdirectory
  (citations actually cited by that paper). A master
  `references.bib` at the `manuscripts/` root collects
  candidate citations used across both.
- **Shared assets** (`data/`, `figures/`, `tables/`) hold
  numerical outputs and figures generated by scripts in `R/`,
  `implementations/`, or `analysis/` that may be reused across
  manuscripts. Paper-specific intermediates stay inside the
  per-paper subdirectory.
- **Reproducibility.** Simulation runs that feed a manuscript
  should be orchestrated from `analysis/` (per project
  convention) and checkpointed to `data/` or `manuscripts/data/`.
  The manuscript `.Rmd` loads finalised results rather than
  re-running simulations inline.

## Relationship to `docs/`

- `docs/` contains the internal technical reports, audit
  notes, and white papers used to develop the arguments.
- `manuscripts/` contains the journal-format distillations
  intended for external submission.

A manuscript may cite specific `docs/` reports as supplementary
material when a journal permits linking to a companion technical
report.

## Splitting out at submission time

If a target journal requires a standalone submission bundle,
use `git subtree split` on the manuscript subdirectory to
create a focused repository preserving the relevant history.
See the `res/04-runin-power-analysis` project for the
in-repository precedent.
