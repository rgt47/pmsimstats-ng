This document indexes the technical documentation for the
\texttt{pmsimstats-ng} simulation framework. Documents are
grouped by their role in two narratives: (a)~the Hendrickson
et al.\ (2020) research program on biomarker-moderated
treatment response in aggregated N-of-1 trial designs, and
(b)~the development path from the original codebase to the
current dual-architecture framework.

\section{Research narrative}

These documents describe the scientific goals, statistical
methods, and simulation results for the Hendrickson et al.\
(2020) power analysis of predictive biomarker-treatment
interactions.

\begin{center}
\begin{tabular}{lp{9cm}}
\toprule
\textbf{Document} & \textbf{Scope} \\
\midrule
\texttt{04} & Revised power analysis with corrected
  covariance structure and carryover model
  (Architecture~B). Includes commit-by-commit
  comparison appendix. \\
\texttt{06} & AR(1) residual correlation: transition
  from \texttt{lmer} to \texttt{nlme} with
  \texttt{corCAR1}. Applies to both architectures. \\
\texttt{07} & Positive definiteness failures in the MVN
  covariance matrix (Architecture~B specific). \\
\texttt{08} & Derivation of the $\lambda_{\text{cor}}$
  BM-BR correlation decay during off-drug periods
  (Architecture~B specific). \\
\texttt{09} & Analysis of the step-function BM-BR
  correlation artifact in the original code
  (Architecture~B specific). \\
\texttt{18} & Response parameter sensitivity (Figure~5):
  original vs.\ revised implementations
  (Architecture~B). \\
\bottomrule
\end{tabular}
\end{center}

\section{Three implementation collections}

The \texttt{implementations/} directory contains three parallel,
complete implementations of the simulation framework, each
supporting different architectures and coding styles:

\begin{center}
\begin{tabular}{lp{8cm}}
\toprule
\textbf{Collection} & \textbf{Scope} \\
\midrule
\texttt{original} & Architecture~B (MVN) only; corrected
  Hendrickson publication code; data.table style. Baseline
  reference implementation. \\
\texttt{original-extended} & Architecture~A + B (both);
  original code extended with
  \texttt{dgp\_architecture} parameter; data.table style.
  Production use. \\
\texttt{tidyverse} & Architecture~A + B (both); complete
  tidyverse reimplementation; tidyverse style. Modern
  alternative. \\
\bottomrule
\end{tabular}
\end{center}

Each collection has a \texttt{README.md} with parameter
documentation, usage examples, and validation notes. All three
accept the same core parameters (\texttt{c.bm},
\texttt{carryover\_t1half}, \texttt{lambda\_cor}); only
\texttt{original-extended} and \texttt{tidyverse} support the
\texttt{dgp\_architecture} parameter for Architecture~A
(mean moderation) vs.\ Architecture~B (MVN).
